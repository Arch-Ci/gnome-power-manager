Performing a release
====================

Follow <http://developer.gnome.org/dotplan/for_maintainers.html>
And <http://live.gnome.org/TwoPointThirteen>

- update configure.in, NEWS, ChangeLog (the "==== <version> ====") to
the right release

- Add translators to NEWS

- Run "make distcheck", correct any problems encountered (don't forget
to update the ChangeLog if you make any changes)

- Commit all the changes

- Copy the resulting tarball to master.gnome.org (scp)
scp gnome-power-manager-*.tar.gz  xxxxxx@master.gnome.org:

- Install the module on master.gnome.org
ssh xxxxxx@master.gnome.org
install-module gnome-power-manager-*.tar.gz

- run "cvs tag <name of tag>".  You can check the past tags by running a
cvs log on the ChangeLog, for example:
export CVS_RSH=ssh
cvs tag GNOME_POWER_MANAGER_2_13_5

- Do a post-release version bump by updating configure.in

- Update jhbuild modulesets

- Notify desktop-devel-list@gnome.org, gnome-doc-list@gnome.org, gnome-i18n@gnome.org

- Done!

Making a Branch
===============

<http://cvs.gnome.org/viewcvs/gep/proposals/gep-4.html?view=markup>
<https://www.cvshome.org/docs/manual/cvs-1.11.19/cvs_5.html#SEC54>

You can create a branch with tag -b; for example, assuming you're in a working copy:

$ cvs tag -b gnome-2-12
